## 本轮关键技术决策

- **本地 `JWT_SECRET` 在 `wrangler.toml` `[vars]`**：dev 环境用占位值（注释明确），prod 用 `wrangler secret put`。`tests/vitest.config.ts` 走独立 bindings，prod 走 secrets，三者解耦。
- **Leaderboard 用 `/api/v1/teams?limit=N` 而非专用端点**：先简单，足够展示。后续可加 KV 缓存。
- **比赛事件时间轴默认折叠**：默认只显示 kickoff/goal/halftime/fulltime/card，可一键展开全部。避免 90 个 shot 事件淹没 UI。
- **OpenAPI 3.1 + Stoplight Elements CDN**：CLAUDE.md §8 的方案，0 打包成本。
- **前端 react-hook-form + zod resolver**：表单校验与后端共享 zod schema（虽暂未共享实例，但结构对齐）。
- **首次部署无需 drizzle-kit generate**：用 hand-written SQL + 顺序单条 INSERT 绕过 D1 限制；drizzle-kit 留作未来需要 schema 演变时再用。
